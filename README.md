# Andrew's Portfolio

A standalone Flask portfolio for showcasing personal projects, experiments, and development work.

## Run locally

1. Create and activate a virtual environment.
2. Install dependencies with `pip install -r requirements.txt`.
3. Copy `.env.example` to `.env` and replace the placeholder values.
4. Run the site with `python app.py`.

## Customize

- Update the `PROJECTS` list in `app.py` with real projects and links.
- Replace the starter biography and toolbox in `templates/about.html`.
- Set your email, GitHub, LinkedIn, title, and public URL in `.env`.
- Adjust the color tokens at the top of `static/css/style.css` to change the visual theme.

Before production, set `FLASK_ENV=production`, use a unique long `FLASK_SECRET_KEY`, and set the public `SITE_URL`.

## Deploy on Render

This project includes a [render.yaml](render.yaml) blueprint for a Render web service.

### 1. Push the repo to GitHub

Render deploys from GitHub, so make sure this project is in a GitHub repository first.

### 2. Create the Render service

1. Sign in to Render.
2. Choose **New +** -> **Blueprint**.
3. Select this repository.
4. Render will detect `render.yaml` and create the web service.

The service uses:

- Build command: `pip install -r requirements.txt`
- Start command: `gunicorn app:app --bind 0.0.0.0:$PORT`

### 3. Set the environment values

In Render, confirm or update these environment variables:

- `FLASK_ENV=production`
- `FLASK_SECRET_KEY=<generated by Render>`
- `SITE_TITLE=Andrew's Portfolio`
- `SITE_DESCRIPTION=Personal projects, experiments, and notes from Andrew.`
- `SITE_EMAIL=<your real email>`
- `SITE_URL=<your Render URL at first, then your real domain later>`
- `GITHUB_URL=<your GitHub profile URL>`
- `LINKEDIN_URL=<your LinkedIn profile URL>`

Important:

- When the service first goes live, set `SITE_URL` to your Render URL, like `https://your-service-name.onrender.com`
- After connecting your custom domain, update `SITE_URL` again to your real domain, like `https://andrewmock.dev`

### 4. Test the live site

Once the deploy finishes, open the Render URL and verify:

- homepage loads
- `/about` loads
- `/projects` loads
- `/games/apollo11`
- `/games/artillery`
- `/games/chess`

### 5. Connect your domain

After the Render version works:

1. Open the service in Render.
2. Go to **Settings** -> **Custom Domains**.
3. Add your domain.
4. Render will give you the DNS records to add at your domain registrar.
5. After DNS verifies, update `SITE_URL` to the final domain.

### Free tier note

Render's free web services can spin down after inactivity, so the first request after a quiet period may take a little longer to load.
